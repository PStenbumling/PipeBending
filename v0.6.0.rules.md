# v0.6.0 Rules

RULE: La hämtas alltid från fittings.json när koppling är vald — aldrig hårdkodat
RULE: La uppdateras automatiskt när Dy ändras och koppling är vald
RULE: Om vald koppling saknar dimension för aktiv Dy sätts La = 0
RULE: La-fältet är inte synligt för användaren — La hanteras internt via state
RULE: defaultCC i fittings.json sätter CC-checkboxen automatiskt vid kopplingsval
RULE: CC-checkboxen kan alltid ändras manuellt av användaren efter auto-sättning
RULE: CC innebär att måttet är till centrum på koppling
RULE: Vid CC ersätts hela slut-formeln: adj[n-1] = −La_slut
RULE: Build appliceras aldrig på slutände när CC är aktivt
RULE: Endast ej-CC slutände använder +build + La
RULE: CC-justering får aldrig ge positiv adj[n-1] — logfel om så sker
RULE: Startände påverkas inte av CC-logik
RULE: Mått till kopplingens öppning innebär att La ska adderas på startände
RULE: Skarvkoppling tolkas som öppningsmått — addsLaAtStart = true
RULE: T-koppling tolkas som CC — addsLaAtStart = false
RULE: addsLaAtStart styr om La adderas på adj[0] — ingen annan logik ändras
RULE: Default (ingen koppling vald) behandlar start som öppning — mbStartAddsLa = true
RULE: Startmått antas vara till kopplingens öppning om inget annat anges
RULE: Om användaren redan räknat med instick i startmåttet kan resultatet avvika med ~La
RULE: Mellansträckor påverkas aldrig av kopplingsval eller CC
RULE: mbComputeMarks() ändras inte
RULE: geBendDeduction() och geParallelOffset() ändras inte
RULE: Kopplingsval visas inline per rad (start-rad + slut-rad) — synligt i båda lägen
RULE: CC-checkbox visas inline på slutraden — alltid synlig
RULE: Default (ingen koppling, CC = av) ger identiskt resultat med v0.5.8
RULE: fittings.json är enda datakälla — inga produktnamn eller dimensioner i index.html
RULE: Inga sidoeffekter i bananböj, S-böj, Svanbock, Takoffset, Takrör
