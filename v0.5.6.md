## Feature Track 2 — Visual Instick in Couplings (Bananböj diagram)

- In `calculateDiagram()` (or equivalent SVG render function for Bananböj):
  - Draw instick zone for T-koppling end: highlighted segment of length `instA` mm inside fitting mouth, distinct color tint from main pipe
  - Draw instick zone for H-stycke end: highlighted segment of length `instB` mm
  - Draw fitting mouth marker line at each transition point (free pipe → inserted pipe)
  - Add labels: `"Instick T-koppling: ${instA}mm"` and `"Instick H-stycke: ${instB}mm"` — whole mm values
  - SVG render order: fittings → instick zones → free pipe path → mouth markers → labels
- Add small legend below SVG:
  - Solid pipe = fri del
  - Highlighted = instick i koppling
  - Marker line = kopplingens mun
  - Hide legend if SVG container width < 300px
- Labels: whole mm, no variable names in user-facing text (`instA` → `"Instick T-koppling"`)
- Instick zones must not visually compete with reference origin — ref-strip is separate from diagram
- **Visual kill-switch rule (critical):** T-koppling end must NEVER be styled with stronger contrast than the reference end (H-stycke). Instick highlight color must be lower contrast than the reference marker. Concrete rule: instick zone opacity ≤ 0.55, reference marker opacity = 1.0. If the SVG renderer uses stroke-width for emphasis, instick strokes must be thinner than the reference arrow stroke.

## Feature Track 3 — Bananböj Simple / Advanced Mode

### Overview

Add a **Enkelt / Avancerat** mode toggle to the Bananböj tab. The toggle controls which input fields are visible. Geometry engine, geometry helpers, and all calculation logic remain unchanged. No new math — only field visibility and a simplified output format.

- **Simple Mode (Enkelt):** Minimal inputs. Angle preset required. D (sidmått) required. M1 and instick use defaults. All other sections hidden. Output is a 3-line summary card.

- Default mode: Simple Mode on first visit. After user switches, persist choice in `sessionStorage` key `banana_ui_mode`.

### JS — `setBananaMode(mode)` function

Add at module level, near other Bananböj state:

```js
let bananaMode = 'simple';

function setBananaMode(mode) {
  bananaMode = mode;
  sessionStorage.setItem('banana_ui_mode', mode);
  document.querySelectorAll('#banana-section .banana-advanced-only').forEach(el =>
    el.classList.toggle('banana-hidden', mode === 'simple'));
  document.querySelectorAll('#banana-section .banana-simple-only').forEach(el =>
    el.classList.toggle('banana-hidden', mode === 'advanced'));
  document.querySelectorAll('.bm-btn').forEach(b =>
    b.classList.toggle('active', b.dataset.bmode === mode));
  const kapaBd = document.querySelector('#kapa-card .kapa-breakdown');
  if (kapaBd) kapaBd.classList.toggle('banana-hidden', mode === 'simple');
  if (mode === 'simple' && lockedAngle === null) {
    setMode('vinkel');
  }
}
```

### JS — `bananaUpdateOffsetPreview()` — preset angle offset preview

Add `<div id="banana-offset-preview" class="hint banana-simple-only"></div>` below the `#djup` hint line in HTML.

Add function:

```js
function bananaUpdateOffsetPreview() {
  const el = document.getElementById('banana-offset-preview');
  if (!el || bananaMode !== 'simple') return;
  if (lockedAngle === null) { el.textContent = ''; return; }
  const D = parseFloat(document.getElementById('djup').value);
  if (isNaN(D) || D <= 0) { el.textContent = ''; return; }
  const geo = geParallelOffset(D, lockedAngle);
  if (geo.L === Infinity) { el.textContent = ''; return; }
  el.textContent = `Vid ${lockedAngle}° och ${Math.round(D)} mm sidmått: snedlängd ≈ ${Math.round(geo.L)} mm`;
}
```

Call `bananaUpdateOffsetPreview()` from:
- `setAngle()` — already fires on preset button click (add one line at end)
- `onLiveInput()` — already fires on `#djup` change (add one line at end)

No new event listeners needed.
