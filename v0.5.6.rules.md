# v0.5.6 Rules — Simple / Advanced Mode + Visual Instick

RULE: Simple Mode never blocks calculation — existing early returns in calculate() are unchanged.
RULE: setBananaMode() never modifies inputs, field values, or calculation logic.
RULE: banana-ref-strip-input is always visible — never assign banana-advanced-only to it.
RULE: banana-ref-strip-result is updated on every calculate() call regardless of active mode.
RULE: kapa value in simple card equals Math.round(kapa) — not a separate calculation.
RULE: M1 value in simple card equals Math.round(M1) — not a default or placeholder.
RULE: bananaUpdateOffsetPreview() uses geParallelOffset() only — no inline trig.
RULE: banana-hidden must never be applied to #preset-row.
RULE: Switching mode mid-session never clears or resets input field values.
RULE: All v0.5.5 warnings (clamp, estimated badge, hierarchy) run unchanged in both modes.
RULE: applyWarningHierarchy() is called last in calculate() in both Simple and Advanced Mode.
RULE: Visual Instick opacity must be ≤ 0.55 — reference marker opacity must be 1.0.
RULE: T-koppling end must never carry an origin label in instick visualization.
RULE: Instick stroke-width must be less than reference marker stroke-width.
RULE: No new geometry functions — geParallelOffset() and geBendDeduction() are the only sources.
RULE: banana-simple-only elements are hidden in Advanced Mode, not removed from DOM.
RULE: banana-advanced-only elements are hidden in Simple Mode, not removed from DOM.
